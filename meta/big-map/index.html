Big Map

Lazy tile reconstruction over very large Coylean universes.

The universe is logically an L × L propagation, but we never hold the whole thing. We sweep it once in K × K blocks, save the horizontal and vertical seams (one per K rows, one per K columns), and discard the interior. Any tile can then be rebuilt from its two adjacent seams in O(K²) time.

Explain Exercise buildScaffold / tile; see seams, ratios, and a tile beside the full reference. Explore Pan and zoom across the universe; arrows zoom-aware, dyadic gridlines thicken with priority. Explore (centered, p≈5) Same explorer, started centered on the origin at cellPx 5 (vector mode) with grid and labels on.

Explore URL params

All optional. Construction params pre-fill the controls; view params place the initial viewport.

  • L int ≥ 2 (def 1024) — per-quadrant extent; total grid (2L-1)²
  • K int ≥ 1 (def 256) — tile block size
  • maxPri int 1–31 (def ⌈log₂ L⌉+1) — priority cap (both axes)
  • maxLatPri int ≥ 1 (def maxPri) — latitude (N–S) cap only
  • maxLongPri int ≥ 1 (def maxPri) — longitude (E–W) cap only
  • hInitCol int (def 1) — h offset in pri(i + hInitCol)
  • vInitRow int (def 1) — v offset in pri(j + vInitRow)
  • seniority v|h (def v) — tie-breaking
  • cx, cy number (def 0) — viewport centre in origin-relative cells
  • px number (def auto fit) — initial cellPx
  • grid, labels 0|1 (def 0) — overlay toggles

Source modules in this directory:

  • scaffold.mjs — buildScaffold({ L, K, ... })
  • tile.mjs — tile(scaffold, k1, k2)
  • selfcheck.mjs — block-wise vs. full propagation
  • render.mjs — canvas helpers shared by the two pages
