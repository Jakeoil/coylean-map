The Ruler Runs Both Ways

So far the ruler is one-sided — defined for n ≥ 1. But it is secretly two-sided, with 0 as an infinite-priority spike at the center and the negative integers mirroring the positive ones. The bridge is the oldest trick in the book: two's complement.

The 8-bit picture

Read an 8-bit pattern the usual way and every place is a positive power of two. Read it in two's complement and the top place flips sign — the high bit is negative. Same bits, new meaning. Take 10101010:

The valuation never looks at the sign. It only reads the lowest set place — and that place is untouched by which weight at the top went negative. So the negatives inherit the exact same ruler the positives have: ρ(−k) = ρ(k).

Counting backwards from zero

In a finite width the integers wrap into a cycle. Go up from 0 and you climb 1, 2, 3, …; go down from 0 and you fall into the top of the cycle:

  • Dyadic. −1 is the all-ones word …11111111; −2 is …11111110; each step down is the two's-complement of the step up. Its lowest set bit — hence ρ₂ — mirrors the positive side exactly.
  • Fibonacci. −1 begins right where the sequence resumes just after the biggest Fibonacci number is hit. Past F_m the ruler resets to 0, 1, 2, 0, 3, … — and that resumed run is the negative axis, read backwards. (Verified: the block after F₈ = 34 is bit-for-bit the base sequence 1, 2, 3, ….)

Either way 0 is the lone infinity: in the 2-adic / Fibonacci-adic limit it is infinitely divisible — no lowest place at all — so its priority is unbounded while every other cell, positive or negative, sits at a finite tick. The finite width is just a simulation of that infinite, two-sided ruler.
