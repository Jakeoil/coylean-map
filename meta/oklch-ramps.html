OKLCH Ramps in Rotation

Ten hue families at 36° increments around the OKLCH wheel, shown with the same five-role ladder plus a gaudy chip on each. Lightness and chroma are constant within each column — only the hue rotates — so the wheel reads as a true perceptual rotation, the way the OKLab space is designed to support.

Per-role spec used across every hue: outline 34% .12 H · shadow 25% .09 H · base 58% .16 H · highlight 74% .12 H · glow 88% .07 H · gaudy 68% .30 H. Hue H steps through 0°, 36°, 72°, 108°, 144°, 180°, 216°, 252°, 288°, 324°.

Names in italics — red, amber, green, blue, violet — mark the five rows that align (within a few degrees) with the theme's existing families in theme.js. Diametrically-opposite rows are complements: rose↔teal (0°↔180°), red↔azure (36°↔216°), amber↔blue (72°↔252°), lime↔violet (108°↔288°), green↔magenta (144°↔324°). Reading the table top-to-bottom traces the full hue wheel once.

Gray — for completeness

Chroma is zero, so the hue parameter has no effect and the ramp is pure lightness. Included here for parity with the colored families; in practice it serves as the achromatic baseline for scaffolding, disabled state, or unclassified values. The gaudy slot is shown at the base value since "gaudy" implies saturation and there's none available.
