OKLCH Color Theme Sketch

A small family-based palette: red and blue stay as anchors, with green, amber, and violet added as balanced companions. Each family has the same roles: outline, shadow, base, highlight, and glow.

Why OKLCH?

RGB and ordinary HSL are convenient, but they do not track perceived brightness very well. OKLCH is better for this kind of rendering because lightness is more visually predictable. A blue at L=58% and a green at L=58% will feel much closer in weight than if you just guessed hex values by eye.

The useful mental model is: oklch(lightness chroma hue). Lightness is brightness, chroma is color strength, and hue is the angle around the color wheel.

Suggested hue families

Family Approx. hue Suggested use
Red 25° energy, conflict, active edge, hot emphasis
Amber / brown-yellow 70° priority, warning, seniority, earth, old state
Green 145° growth, valid path, confirmation, live state
Blue 260° structure, stable geometry, base field, cool emphasis
Violet 305° meta layer, alternate propagation, exotic/special state
Gray — achromatic baseline, disabled/inactive, scaffolding, unclassified state (chroma = 0, hue irrelevant)

Brown is usually not a separate hue. It is dark, muted orange/amber. So an amber family gives you yellow highlights and brown shadows without adding a sixth competing color.

Complementarity

OKLCH stands for Oklab Lightness, Chroma, Hue. The practical form is oklch(L C H): lightness, color intensity, and hue angle. It is like HSL, but the lightness behaves much more like human perception.

A simple complement is roughly hue + 180°. So blue around 260° points toward amber around 80°. Red around 25° points toward cyan/teal around 205°. Green around 145° points toward magenta/violet around 325°.

But in rendering, exact complements can be too loud. Usually you want near complements: blue vs amber, red vs greenish-cyan, green vs violet. These are still oppositional, but less like a warning sign. See the full hue wheel laid out as ramps → — diametrically-opposite rows are the true 180° complements.

Practical rule

Let hue carry category. Let lightness carry depth. Let chroma carry emphasis. That prevents the palette from becoming decorative noise.

For example: use the darker members for outlines and shadows, the base member for ordinary filled regions, the highlight for selected edges, and the glow for halos, gradients, or soft background effects.
