How to Draw a Coylean Square

A Coylean square is a compass-and-straightedge drawing: vertical and horizontal passes across a square, each lifting and dropping the pencil whenever it crosses a line already on the paper. Here the passes are read straight off a finished propagation — drawn in priority order, most senior line first — so you can watch the construction unfold and see that the algorithm is just its bookkeeping shadow. Switch to Universe to draw the whole plane out of the central ∞-cross, or Map for an arbitrary south × east patch.

From a finished propagation

Build a propagation (engine Propagation.fromUniverseExtents, anchored at the origin). Each column carries a ruler priority colPriority[i] for its vertical (down) line, and each row a rowPriority[j] for its horizontal (right) line — the 2-adic valuation of the offset index. Then draw by priority, most senior first:

  • level = max(colPriority, rowPriority);
  • for every column at that level, run a vertical pass; for every row at that level, a horizontal pass (the seniority toggle picks which half goes first);
  • drop a level and repeat down to 0.

Each pass is the same pencil-toggle geometry: travel the line, flip the pencil at every existing perpendicular segment you cross, and draw only the pencil-down stretches. Those stretches are exactly the maximal on-runs of the finished propagation's downMatrix / rightMatrix, so the drawing is the propagation, segment for segment — verified in Node for both modes, all four anchors, both seniorities, and non-square / shifted lattices. A single pass can leave several disjoint pieces.

Square vs Universe vs Map

Square mode integrates a 2ⁿ × 2ⁿ patch — the genuine self-similar Coylean square — including the western / northern context column only where that anchor needs its ∞ frame there (i.e. where the offset is 1). Universe mode instead puts the origin in the middle: a symmetric 2ⁿ-radius window whose two ∞ axes cross at mid-board, with the four quadrants radiating out. It draws from that cross outward — each vertical line filling down then up, each horizontal right then left, columns and rows walked centre-first — so you watch the map grow into the plane rather than sweep in from a corner. Map mode takes an arbitrary south × east extent and needs no context cell (north and west extents come later). The orientation triplet is the anchor: ↔ longitude and ↕ latitude each toggle the dyadic offset ∈ {0, 1} (the four anchors SE / SW / NE / NW), and ⤢ seniority is the V↔H tie-break — the backslash dual.

Why the algorithm is a shadow of this. The propagation rule — priority, reaction, propagation — just records, cell by cell, whether the pencil happened to be down on each edge. The 2-adic priority that breaks its ties is exactly the seniority of these passes; one reaction is one crossing.
