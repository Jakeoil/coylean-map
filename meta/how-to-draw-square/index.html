How to Draw a Coylean Square

The Coylean square is, first of all, a compass-and-straightedge drawing: you make vertical and horizontal passes across a square, and each pass lifts and drops the pencil whenever it crosses a line already on the paper. Watch the construction unfold pass by pass. The propagation algorithm elsewhere on the site is just a bookkeeping shadow of this geometry.

The geometric rule

Start with the bare square ABCD — that is order 0. Each later order k adds 2k-1 vertical passes and the same number of horizontal passes, placed at the new odd dyadic fractions 1⁄2k, 3⁄2k, …, (2k−1)⁄2k. By convention all of an order's verticals are drawn first, then its horizontals — so the horizontals already "see" the verticals of their own order.

A vertical pass at column x:

  • start at the top edge, pencil down;
  • travel straight down to the bottom edge;
  • every time you cross a horizontal segment that actually exists at that point, flip the pencil (down → up, up → down);
  • draw only the stretches travelled pencil-down.

A horizontal pass is the mirror image: travel left to right, flipping the pencil at every existing vertical segment it crosses. Crossings count only where the perpendicular line is really present — there is no full grid underneath, just the segments drawn so far. A single pass can therefore leave several disjoint pieces.

Why the algorithm is a shadow of this. The propagation rule — priority, reaction, propagation — just records, cell by cell, whether the pencil happened to be down on each edge. Tie-breaking by 2-adic priority is exactly the seniority of these dyadic passes; the reaction box is one crossing. Run this page to order 3 and the propagation map is pixel-identical (verified in Node against coylean-core).
